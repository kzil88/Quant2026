# ML 机器学习策略原理与实现

## 核心思想

**让机器自己找规律。** 把因子值当输入特征、未来收益当预测目标，用机器学习模型（LightGBM / XGBoost）训练，预测哪些股票会涨。

和多因子策略的区别：多因子是人工定权重（EP 40%、ROE 30%...），ML 策略让算法自己从数据中学习最优的因子组合方式，包括非线性关系和因子交互。

---

## 实现流程

### 第1步：构建训练集（fit 阶段）

```
特征 X = 每只股票在每个时间点的因子值矩阵（14个因子）
标签 y = 该股票未来 N 天的实际收益率
```

举例：2024年1月15日，茅台的 EP=0.025、ROE=0.30、RSI=62...（14个特征），未来20天实际涨了3%（标签=0.03）。

把所有股票、所有日期的数据拼在一起，就是训练集。

### 第2步：训练模型

项目支持两种梯度提升树模型（实现见 `quant2026/strategy/ml_model/strategy.py`）：

**LightGBM（默认）：** 微软开源，速度快，内存省，适合大数据集
**XGBoost：** 经典选手，稳定性好

默认参数：
- 200棵树（`n_estimators=200`）
- 最大深度5层（`max_depth=5`）
- 学习率0.05（`learning_rate=0.05`）

模型学到的是：什么样的因子组合预示着未来收益高。比如它可能自动发现"EP高 + ROE高 + RSI<30 的股票涨得好"——这种非线性组合是线性加权打分学不到的。

### 第3步：预测打分（generate 阶段）

拿当天所有股票的因子值喂给训练好的模型，输出每只股票的预测收益率。

```python
preds = model.predict(factor_matrix)   # 每只股票一个预测收益
top = preds.nlargest(top_n)            # 取预测收益最高的 N 只
```

### 第4步：特征重要性

训练完成后自动记录每个因子的重要性排名，知道模型最看重哪些因子。

---

## 和多因子策略的对比

| 维度 | 多因子（线性打分） | ML 策略 |
|------|-------------------|---------|
| 因子组合方式 | 人工定权重，线性加权 | 模型自动学习，可捕捉非线性 |
| 可解释性 | 高（权重清楚） | 中（有 feature importance 但不直观） |
| 过拟合风险 | 低 | 高（需要 Walk-Forward 验证） |
| 数据需求 | 少 | 多（样本太少模型学不好） |
| 适应性 | 权重固定 | 可重新训练适应市场变化 |

---

## 防止过拟合

ML 策略最大的风险是**过拟合**——模型记住了历史噪音而不是真规律。项目的应对措施：

1. **Walk-Forward 验证**：滚动窗口训练+测试，模拟真实场景
2. **控制模型复杂度**：树深度限制5层，防止记忆噪音
3. **训练样本检查**：少于10个样本直接拒绝训练

## 参数说明

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `model_type` | `"lightgbm"` | 模型类型，可选 `"xgboost"` |
| `forward_days` | 20 | 预测未来多少天的收益 |
| `train_window` | 252 | 训练窗口（约1年交易日） |
| `top_n` | 30 | 选前多少只股票 |
| `model_params` | 见默认值 | 可覆盖模型超参数 |

---

## 适用场景与局限

**适合：** 因子之间存在复杂交互关系时，线性模型无法捕捉的 alpha
**局限：**
- 需要大量历史数据（至少1年以上）
- 过拟合风险高，必须配合严格的样本外验证
- 市场 regime 变化时，历史学到的规律可能失效
- 黑箱特性，出问题时不容易定位原因
