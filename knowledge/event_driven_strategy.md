# 事件驱动策略原理与实现

## 核心思想

**盯事件，不盯K线。** 当公司发生特定事件（财报超预期、大宗交易、股东增减持）时快速建仓，吃事件带来的短期价格冲击。

类比：不猜股票会涨还是跌，而是蹲在"公告板"前面，一看到好消息就买、坏消息就避。信息驱动，和技术面无关。

---

## 三类事件及打分逻辑

项目监控三类事件（实现见 `quant2026/strategy/event_driven/strategy.py`），各有权重：

### 1. 财报超预期（权重 50%）

```
得分 += surprise_pct × earnings_weight
```

- `surprise_pct`：实际利润超出市场预期的百分比
- 超预期越多 → 得分越高 → 越值得买

**原理：** 财报超预期后，分析师会上调目标价，资金会涌入，股价通常有一波上涨（学术上叫 PEAD，Post-Earnings Announcement Drift）。

### 2. 大宗交易溢价（权重 30%）

```
得分 += premium_pct × block_trade_weight
```

- `premium_pct`：大宗交易价格相对于市场价的溢价率
- 有人愿意溢价大笔买入 → 说明看好这只股票

**原理：** 大宗交易是机构之间的大额买卖。如果成交价高于市场价（溢价），说明买方急于建仓，是积极信号。

### 3. 股东增减持（权重 20%）

```
增持 → 得分 += |change_pct| × shareholder_weight
减持 → 得分 -= |change_pct| × shareholder_weight
```

- 大股东/高管增持 → 加分（他们最了解自己公司）
- 大股东/高管减持 → 减分（可能觉得股价高了）

**原理：** 内部人最了解公司真实状况。高管自掏腰包增持通常是强烈看好信号。

## 实现流程

### 1. 设定回看窗口

默认 `lookback_days=30`，只关注最近30天内的事件。太久之前的事件价格已经反映了。

### 2. 收集事件

通过 `EventCollector` 分别获取三类事件数据：
- 财报超预期数据
- 大宗交易数据
- 股东增减持数据

### 3. 按股票汇总得分

同一只股票可能同时有多个事件（比如财报超预期 + 高管增持），分数会叠加。多重利好叠加的股票得分最高。

### 4. 排序输出

没有事件的股票得分为0。最终按总分排序，高分的优先买入。

---

## 参数说明

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `earnings_weight` | 0.5 | 财报超预期的权重（最重要） |
| `block_trade_weight` | 0.3 | 大宗交易溢价的权重 |
| `shareholder_weight` | 0.2 | 股东增减持的权重 |
| `lookback_days` | 30 | 事件回看窗口（天） |

权重设计体现了信号强度的先验判断：财报是最硬的数据 > 大宗交易是真金白银 > 增减持有时是例行计划。

## 适用场景与局限

**适合：**
- 信息不对称较大的市场（A股散户多，信息扩散慢）
- 和其他策略互补——其他策略看量价，事件驱动看基本面变化
- 财报季（密集发布期）效果最明显

**局限：**
- 依赖事件数据的及时性和质量
- 非财报季事件稀疏，策略可能长期无信号
- "超预期"本身依赖一致预期数据，预期数据有偏
- 减持不一定是利空（可能是个人资金需求），增持也不一定涨

---

## 和其他策略的关系

事件驱动策略通常作为**辅助策略**和其他策略混合使用：

- 多因子策略提供常态化的选股能力
- 事件驱动在特定时间点提供"催化剂"信号
- 配合使用 = 基本面好（多因子高分）+ 刚发生利好事件 → 双重确认
