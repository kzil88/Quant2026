# 统计套利（配对交易）策略原理与实现

## 核心思想

**不赌涨跌，赌关系。** 找两只历史走势高度相关的股票，当它们的价差异常偏离时，买入落后的、卖出领先的，等价差回归正常获利。

类比：两个同班同学历史考试成绩一直很接近，这次A考了90分B只考了70分。统计套利赌的是"下次B会追回来"，而不是赌B一定考多少分。

---

## 实现流程

### 第1步：寻找协整配对

不是随便两只股票都能配对。项目用 **Engle-Granger 两步法**检验协整关系（实现见 `quant2026/strategy/stat_arb/cointegration.py`）：

**什么是协整？** 两个序列虽然各自随机游走（无法预测），但它们的差值（价差）是均值回归的。好比两条狗各自乱跑，但被一条绳子拴着——不会离得太远。

检验步骤：
1. **OLS 回归**：用 B 的价格去拟合 A 的价格，得到对冲比率（hedge ratio）
2. **协整检验**：用 `statsmodels.coint()` 检验价差是否平稳，p < 0.05 才算协整
3. **遍历所有组合**：N 只股票两两配对检验，按 p 值从小到大排序
4. **取最好的**：保留 max_pairs 对（默认10对）

### 第2步：计算价差 Z-score

对每对协整股票：

```
价差 = price_A - hedge_ratio × price_B
Z-score = (价差 - 均值) / 标准差
```

Z-score 衡量当前价差偏离历史正常水平多远。

### 第3步：产生交易信号

| Z-score | 操作 | 逻辑 |
|---------|------|------|
| Z < -2.0 | 买入 A | A 相对于 B 被低估，价差偏低 |
| Z > +2.0 | 买入 B | B 相对于 A 被低估，价差偏高 |
| \|Z\| < 0.5 | 平仓 | 价差回归正常 |

**A股限制：** 不能做空，所以只能做多被低估的一方（long-only variant）。

### 第4步：打分

股票的得分 = 所有涉及它的配对中 |Z-score| 的累加。出现在越多偏离的配对中、偏离越大，得分越高。

---

## 半衰期（Half-life）

项目还计算了价差的**均值回归半衰期**——价差回归到一半需要多少天。

用 AR(1) 模型（Ornstein-Uhlenbeck 过程）：

```
half_life = -ln(2) / ln(ρ)     # ρ 是 AR(1) 系数
```

半衰期越短 = 价差回归越快 = 策略周转越快。

---

## 定期重新计算

协整关系不是永恒的——两家公司可能因业务变化而走势分离。所以每隔 `recalc_interval`（默认60个交易日）重新检验一次配对。

---

## 参数说明

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `lookback` | 120 | 协整检验的回看窗口（交易日） |
| `entry_zscore` | 2.0 | Z-score 超过此值开仓 |
| `exit_zscore` | 0.5 | Z-score 低于此值平仓 |
| `max_pairs` | 10 | 最多持有多少对 |
| `recalc_interval` | 60 | 每隔多少天重新检验配对 |

---

## 适用场景与局限

**适合：** 市场中性（不依赖大盘涨跌），适合震荡市和下跌市
**局限：**
- A股不能做空，只能做 long-only，少了一条腿
- 协整关系可能突然断裂（"regime change"），价差不回来了
- 需要足够多的股票才能找到好的配对
- 回测中表现 6.74% 年化、-16.95% 最大回撤，在6个策略中表现最保守
